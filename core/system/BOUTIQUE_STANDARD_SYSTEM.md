# BOUTIQUE STANDARD — SYSTEM DEFINITION

## 1. PURPOSE

Boutique Standard is a structured hospitality experience intelligence system.

It produces:
- Guest experience audits
- Case reports
- Web summaries
- PDF reports

It does NOT generate marketing copy.

---

## 2. CORE PRINCIPLE

All outputs must be derived from structured data.

No interpretation beyond defined fields is allowed.

---

## 3. INFORMATION MODEL

All properties are represented as:

- Property metadata
- Guest journey stages
- Scores (0–10)
- Observations
- Memory indicators
- Assets (images + notes)

---

## 4. OUTPUT TYPES

### WEB OUTPUT
- compressed
- modular
- scan-friendly
- template-based
- no narrative expansion

### PDF OUTPUT
- full narrative
- editorial pacing
- full context allowed
- page structured

---

## 5. CONTENT RULES

- No invention of new insights
- No poetic expansion
- No marketing language
- No duplication across sections
- Every section must serve one function only

---

## 6. DESIGN PRINCIPLE

Structure is fixed. Content is variable.

Meaning:
- layout never changes
- templates never change
- only data changes

---

## 7. REUSABLE BLOCKS

- Hero Block
- Method Block
- Score Block
- Journey Block
- Insight Block
- Case Block
- Review Block

---

## 8. TONE

Expert observational tone.

Characteristics:
- precise
- controlled
- minimal emotional language
- no storytelling expansion

---

## 9. CLAUDE RULE

Claude acts only as a renderer of structured data.

It must not re-evaluate or reinterpret meaning.

---

## 10. VS CODE RULE

All system files are source-of-truth.

Chat is not source-of-truth.
